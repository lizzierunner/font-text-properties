Font Properties

Although some of the properties sound similar to text properties, think of font properties as those that will change the style of the font on the page — its size, the family, etc.

Font Properties Covered in Your zyBook

font-family
Defines the fonts you want to use. Always list at least two that look similar, so if one fails to load, the fallback displays.
font-size
Controls the size of the text. Browsers apply defaults, so you should be able to spot headings vs. paragraphs even without styling.
font-weight
Makes text bolder or lighter. Headings are usually bold, while body text is normal weight.
font-style
Allows italic, oblique, or normal styles.
font-variant
Specifies variants such as small caps or uppercase letters.
font
Shorthand property to set style, variant, weight, size/line-height, and family all at once.

Using Web Fonts

You can host fonts yourself or link to fonts hosted by services like Google Fonts. Often, the heading font is different from the body font.

In Chrome, go to the font "Seymour One" on Google Fonts. Select the style, copy the link code into your HTML head above your stylesheet, then apply it in your CSS.

Font Sizing

When sizing text, keep these tips in mind:

  1. Use relative units like rem or em so users can scale text.
  2. Ensure body text is at least 1rem (~16px) for readability and accessibility.

Text Properties

Text properties control how text is displayed on the page and can change alignment, decoration, case, and spacing.

Text Properties Covered in Your zyBook

text-align
Controls alignment: left, right, center, or justified.
text-decoration
Controls line style, thickness, color, and position (underline, overline, etc.).
text-transform
Changes case: uppercase, lowercase, or capitalize.
text-indent
Indents the first line of a paragraph. Helpful for readability when scanning long text.

Line Height and Width for Text – Best Practices

These settings improve readability and accessibility:

  • Use a line height greater than 1 — 1.5 is common.
  • Limit paragraph width to 45–90 characters for easier reading (55–65 is ideal).
  • Use the ch unit to limit width based on character count.
